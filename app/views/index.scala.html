This is a sample application demonstrating the use of Scalaz Monad Transformers to avoid deeply nested code when dealing with nested containers like `Future[Option[A]]`. More background material can be found in this presentation and the GitHub repo that goes with it.

The versions below all behave the same.

Version 1

  • No username in the query string
  • Unknown username
  • User with invalid email address
  • User with blacklisted email address
  • User with everything OK

Version 2

  • No username in the query string
  • Unknown username
  • User with invalid email address
  • User with blacklisted email address
  • User with everything OK

Version 3

  • No username in the query string
  • Unknown username
  • User with invalid email address
  • User with blacklisted email address
  • User with everything OK
